# 固件说明

目标主控：CH552G。

固件行为：

1. 枚举为 USB CDC 虚拟串口设备。
2. 初始化红、黄、绿三个 GPIO 为输出。
3. 复位后默认三颗灯全灭。
4. 从虚拟串口读取一行 ASCII 命令。
5. 按 `docs/hardware/serial-protocol.md` 解析命令。
6. 返回 `OK`、当前状态，或错误信息。

## 状态流程草案

```text
USB 复位
  -> 初始化时钟和 USB 设备
  -> 初始化 GPIO 输出为低电平
  -> 等待串口命令行
  -> 解析命令
  -> 更新三颗灯状态
  -> 返回执行结果
  -> 等待下一条命令
```

## 构建工具

工具链还没有锁死。实用选择有：

- 沁恒官方 8051 开发流程。
- 基于 SDCC 的 CH55x 开发流程。

第一目标是先找到稳定的 CDC 示例工程；只要串口协议不变，底层工具链后续可以替换。
